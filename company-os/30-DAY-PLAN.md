# 30-Day Plan — Putting the Company OS to Work

The goal is not to build everything in week one. It is to make one repeated task reliable, then add the next. ModernFlow already has a defined offer, pricing, and a live site — so this plan skips validation and goes straight to operations.

## Week 1 — Foundation (about 2 hours total)

- [ ] Create the Claude Project `ModernFlow AI Operations`
- [ ] Paste `project-instructions.md` into the Project instructions
- [ ] Upload the four `01_Context/` files
- [ ] Fill in the MISSING sections of `01-company.md` (current goals) and `04-operating-context.md` (this week's outcome, active builds, pipeline)
- [ ] Resolve the client-count discrepancy (site says both "500+ clients" and "150+ contractors") and record the verified number
- [ ] Connect calendar and email; run one read-only test
- [ ] Run the **morning brief** template manually each weekday; correct its classifications — the corrections are the training data

## Week 2 — Sales department

- [ ] Run **prospect-qualification** on 5 real contractors (from GHL, referrals, or local search)
- [ ] Use **reply-prep** on every meaningful prospect thread
- [ ] After the next strategy call, drop the notes in `00_Inbox/` and run the **proposal** template; compare against how you'd have written it
- [ ] Add real call phrases to `02-customer.md` — replace the FAQ placeholders with words actual prospects used
- [ ] Keep running the morning brief daily

## Week 3 — Operations rhythm + first Skill

- [ ] Run the first **Friday weekly review**; update `04-operating-context.md` from it
- [ ] Morning brief classifications should now be mostly right — zip `06_Skills/morning-brief/` and install it as a Skill (`/morning-brief`)
- [ ] Connect GoHighLevel (read-only usage first): pipeline summaries into the morning brief
- [ ] Start `05_Logs/`: one line per day on what Claude produced and what you corrected

## Week 4 — Content + remaining Skills

- [ ] Run the **content-interview** workflow once; publish one piece (publishing stays human)
- [ ] Paste 500–1,000 words of your own writing into `03-voice-and-standards.md` as the real voice sample
- [ ] If proposals have gone well manually 2–3 times, install `/proposal-builder`
- [ ] Install `/weekly-review` after the second manual Friday review
- [ ] Only now consider scheduling: the morning brief as a recurring task, with the read-only rules intact

## Standing rules for the whole month

- Nothing reaches a customer without passing through `04_Review` and your eyes
- No new integration until the previous workflow has worked for a week
- If a task repeats three times, it becomes a template; if a template runs clean for two weeks, it becomes a Skill; only Skills that are read-only get scheduled
- Pricing, discounts, refunds, contracts, and ad spend are never delegated
