# Purpose

Create a clear, accurate proposal for a qualified contractor prospect after a strategy call.

# Required inputs

- Strategy-call notes (from 00_Inbox)
- 01-company.md (approved pricing and guarantee terms)
- Customer name and trade
- Desired outcome in the customer's words
- Recommended plan tier (Starter / Growth / Elite) with reason

If any required input is absent, stop and list it.

# Process

1. Extract the contractor's problem using their own language from the call.
2. Identify the desired business outcome (jobs booked, not features).
3. Check the recommended scope against the plan tiers in 01-company.md — nothing outside a tier's inclusions without explicit founder approval.
4. Use only approved pricing: Starter $1,000 + $97/mo; Growth $2,000 + $297/mo; Elite $3,000 + $697/mo.
5. If paid ads were discussed, state the 90-day minimum commitment explicitly.
6. Select only relevant, verified proof (build time 2–3 weeks, sub-5-minute response, 30-day guarantee, client data ownership).
7. Draft the proposal: Situation → Outcome → Recommended plan → Deliverables → Timeline → Investment → Client responsibilities → Guarantee → Next step.
8. Check for unsupported promises (job counts, revenue figures, ads results inside 30 days).
9. Save to 04_Review/proposal-[client]-[YYYY-MM-DD].md.

# Quality checks

- One page where possible
- No invented figures or client results
- No generic company introduction
- No vague next step — one action with a date
- No promise outside the approved plan scope
- Every placeholder removed or marked MISSING

# Approval boundary

Never send the proposal.
Never alter approved pricing.
Never add a discount.
Never promise a build faster than 2–3 weeks.
