# Purpose

Find slippage, bottlenecks, and repeatable work; keep 04-operating-context.md true.

# Required inputs

- This week's calendar
- Sent and received customer email
- Files created in 03_Outputs and items in 04_Review
- 04-operating-context.md and 05_Logs
- Current pipeline (GHL export or the pipeline section of 04-operating-context.md)

If any input is absent, note it and proceed with what exists.

# Process

1. Compare planned work (last week's priorities) to what shipped.
2. Summarize pipeline movement: calls booked, proposals sent, plans closed by tier, builds started/completed.
3. List commitments created this week and commitments at risk — especially builds nearing the 2–3 week promise and clients in the 30-day guarantee window.
4. For every missed item, assign exactly one cause: unclear decision / missing information / unrealistic capacity / external dependency / execution failure / unnecessary work.
5. Identify work repeated 3+ times that should become a template or Skill.
6. Identify work that should stop.
7. Propose three priorities for next week.
8. Draft the update to 04-operating-context.md as a proposed diff.
9. Save to 03_Outputs/weekly-review-[YYYY-MM-DD].md.

# Quality checks

- No motivational advice
- Every "at risk" item names a date and an owner action
- Causes are one of the six categories, not narratives

# Approval boundary

Never apply changes to 04-operating-context.md without approval.
Never message customers.
Never close, move, or edit pipeline records in GoHighLevel.
