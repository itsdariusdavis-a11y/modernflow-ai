# /architecture/ — Layer A (SOPs)

Technical Standard Operating Procedures in markdown. One file per system or
tool-chain. Each SOP contains: goal, inputs, tool logic, edge cases, and
lessons learned from production failures (self-annealing loop, Phase T).

**Golden Rule:** if logic changes, update the SOP here *before* changing the
code in `/execution/`.

_Empty until Phase A — the Blueprint must be approved first._
